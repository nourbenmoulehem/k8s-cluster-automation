# k8s-cluster-automation

Automated Kubernetes cluster deployment on OpenStack using Ansible. Provisions infrastructure via OpenStack Heat templates and configures a full K8s cluster (1 master + 2 workers) with Calico CNI, a monitoring stack, and a sample application.

## Architecture

```
OpenStack
├── Heat Stack
│   ├── Master Node    (k8s-master flavor)
│   ├── Worker1        (ubuntu-small flavor, dynamic via Heat)
│   └── Worker2        (ubuntu-small flavor, static)
└── Security Group     (SSH, K8s API, etcd, kubelet, NodePort, Calico VXLAN)
```

**Network:** Calico CNI with VXLAN mode (required for OpenStack compatibility, avoids IPIP encapsulation issues)

**Pod CIDR:** `10.244.0.0/16`

## Prerequisites

- OpenStack environment with Heat support
- `openstack` CLI configured (sourced RC file)
- Ansible installed on the controller node
- SSH key pair (`~/.ssh/id_ed25519`) added to OpenStack as `mykey`
- Ubuntu 24.04 image available in OpenStack as `ubuntu-24.04-server`

## Quick Start

```bash
./deploy-openstack.sh
```

The script will:
1. Delete any existing `k8s-production` Heat stack (entire cluster is ephemeral)
2. Create a new Heat stack provisioning all 3 nodes via `k8s-full-cluster-template.yml`
3. Extract floating IPs from the OpenStack servers
4. Generate `inventory-openstack.yml` dynamically
5. Wait for all nodes to be SSH-accessible
6. Run the full Ansible playbook

## Manual Deployment

```bash
# Generate/edit inventory manually
vim inventory-openstack.yml

# Run the full playbook
ansible-playbook -i inventory-openstack.yml site.yml

# Or run specific plays with tags
ansible-playbook -i inventory-openstack.yml site.yml --tags "common"
ansible-playbook -i inventory-openstack.yml site.yml --tags "k8s-master"
ansible-playbook -i inventory-openstack.yml site.yml --tags "monitoring"
```

## Project Structure

```
.
├── deploy.sh                    # Full orchestration script (Heat + Ansible)
├── deploy-openstack.sh          # OpenStack-only deployment
├── site.yml                     # Main playbook
├── inventory.yml                # Ansible inventory (auto-generated by deploy.sh)
├── inventory-openstack.yml      # OpenStack dynamic inventory config
├── heat/
│   ├── k8s-full-cluster-template.yml   # Full cluster Heat template
│   └── worker-instance.yml             # Single worker Heat template
└── roles/
    ├── common/                  # Base node prep (swap, kernel modules, sysctl)
    ├── container-runtime/       # containerd installation and config
    ├── kubernetes/              # kubeadm, kubelet, kubectl (v1.30)
    ├── k8s-master/              # kubeadm init, Calico CNI, storage provisioner
    ├── k8s-worker/              # kubeadm join
    ├── monitoring/              # Prometheus + Grafana via Helm
    ├── nestie-app/              # Sample 3-tier app (MySQL, Backend, Frontend)
    └── claude-observer/         # (experimental) Claude-based cluster observer
```

## Roles Overview

| Role | Runs On | Description |
|------|---------|-------------|
| `common` | All nodes | Hostname, /etc/hosts, swap disable, kernel modules (overlay, br_netfilter), sysctl |
| `container-runtime` | All nodes | Install containerd, enable SystemdCgroup |
| `kubernetes` | All nodes | Install kubelet/kubeadm/kubectl v1.30, hold versions |
| `k8s-master` | Master only | kubeadm init, kubectl config, Calico VXLAN, local-path storage, join command |
| `k8s-worker` | Workers only | kubeadm join using generated join command |
| `monitoring` | Master | Helm, kube-prometheus-stack (Prometheus + Grafana) |
| `nestie-app` | Master | Deploy MySQL + backend + frontend to the cluster |

## Inventory Format

Generated automatically by `deploy-openstack.sh` into `inventory-openstack.yml`:

```yaml
all:
  vars:
    ansible_ssh_private_key_file: ~/.ssh/id_rsa
    ansible_ssh_common_args: '-o StrictHostKeyChecking=no'
    ansible_python_interpreter: /usr/bin/python3
  children:
    k8s_cluster:
      children:
        k8s_master:
          hosts:
            master:
              ansible_host: <master-floating-ip>
              ansible_user: ubuntu
              internal_ip: <master-internal-ip>
        k8s_workers:
          hosts:
            worker1:
              ansible_host: <worker1-floating-ip>
              ansible_user: ubuntu
              internal_ip: <worker1-internal-ip>
            worker2:
              ansible_host: <worker2-floating-ip>
              ansible_user: ubuntu
              internal_ip: <worker2-internal-ip>
```

## Verification

```bash
# SSH to master
ssh ubuntu@<master-floating-ip>

# Check cluster
kubectl get nodes -o wide
kubectl get pods -A
kubectl get pods -n monitoring

# Or directly from the controller
ssh ubuntu@<master-ip> 'kubectl get nodes'
ssh ubuntu@<master-ip> 'kubectl get pods -A'
```

## Heat Templates

- **`k8s-full-cluster-template.yml`** - Provisions all 3 nodes (master + 2 workers) with floating IPs — used by `deploy-openstack.sh`
- **`worker-instance.yml`** - Provisions only worker1 (used by `deploy.sh` for the hybrid static/dynamic setup)

Key parameters (with defaults):
- `key_name`: `mykey`
- `image`: `ubuntu-24.04-server`
- `network`: `selfservice`
- `external_network`: `provider`

## Stack Management

```bash
# View stack status
openstack stack show k8s-production

# View events
openstack stack event list k8s-production

# Destroy entire cluster
openstack stack delete k8s-production
```

> The cluster is fully ephemeral — deleting the stack removes all nodes.
